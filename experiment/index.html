JavaGrok Experiment

Hello! Thanks for participating in this experimental evaluation of our JavaGrok tool. We've provided these instructions to give you a bit of background and guide you through the programming tasks we'd like you to complete.

Introduction

JavaGrok is a tool that enhances Javadoc documentation with the results of a variety of program analyses in hopes of giving you useful additional information on how to use a library with which you may not be familiar.

The enhancements to the Javadoc take the form of annotations added to the method documentation. Each annotation communicates some property about the method in question. The exact set of annotations added by JavaGrok is described below, but first we'll give you a concrete example.

Annotation Example

The following is an example of a method augmented by JavaGrok annotations:

getViewBounds

@NonUniqueReturn
public Rectangle getViewBounds()
Get the bounds of the viewport, in media coordinates. For the base MediaPanel, this will always be (0, 0, width, height).

This method has the @NonUniqueReturn property which indicates that a reference to the object returned by the method is retained by the object on which the method was called. In this case, we can interpret that to mean that the Rectangle returned by getViewBounds() has not been defensively copied and as a result should probably not be modified by the caller.

Experiment Procedure

Your mission, should you choose to accept it, is to complete a series of programming tasks that make use of a 2D graphics and animation library with which we expect you are not familiar. We have provided Javadoc documentation for the library (augmented by JavaGrok) as well as English language descriptions of the tasks you are to perform. You will likely need to refer to the library documentation to correctly complete these tasks. We have also provided source to the library to which you may refer if you find the documentation to be insufficient for completing your tasks.

Checklist
We have also provided a checklist on which we request that you record your experiences while completing these programming tasks. The checklist is meant to give us some quantitative feedback on how frequently you find the documentation augmentations to be useful, as well as how frequently you find that you need to refer to the library source code because the Javadoc and annotations do not provide sufficient information. Please try to update the checklist as you proceed through the tasks.

Tasks and Build System

Task Source Files
Each task has an associated source file in which the code for that task should be placed. For example, code for task 1 should be added to src/experiment/Task1.java. Comments in each task source file describe the goal of the task and provide indications of which classes from the graphics library will be useful in completing the task.

Build System
A build system has been provided for compiling and running the tasks. The ant build tool is used to compile and run the tasks. A separate target exists to compile and run each task. If you invoke the build system from the command line, you would do so as follows: 

% ant task1

You may opt instead to use a development environment such as Eclipse. .project and .classpath files have been provided that should allow you to simply import the experiment directory directly into Eclipse and write and execute the tasks from there. Using any other IDE is also fine.
